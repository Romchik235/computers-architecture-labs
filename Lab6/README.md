## Мета роботи
Ознайомитись із механізмами раннього виявлення помилок у модулях ядра Linux та дослідити поведінку ядра під час аварійних ситуацій (BUG, OOPS). Вивчити роботу `BUG_ON()`, NULL pointer dereference, а також способи діагностики помилок.

---

## Опис модулів

### **hello1.c**
- Реалізує функцію `print_hello()`.
- Веде підрахунок викликів.
- На 5-му виклику навмисно створюється помилка, встановивши один із вказівників у `NULL`.
- Це викликає **kernel OOPS** (NULL pointer dereference).

### **hello2.c**
- Імпортує символ `print_hello()` з hello1.
- Має параметр `count`, який визначає кількість викликів.
- Якщо `count` недопустимий — виконується `BUG_ON(count < 0 || count > 10)`, що викликає **kernel BUG**.
- При нормальному значенні `count` модуль викликає `print_hello()` і виводить `"Hello, world!"`.
